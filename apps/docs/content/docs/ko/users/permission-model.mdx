## 왜 권한 모델이 중요한가

운영 사고의 상당수는 기능 부족보다 권한 과다에서 발생합니다.

`outline-mcp`는 클라이언트별 액션 권한을 명시적으로 제한할 수 있으므로, 역할에 맞는 최소 권한 원칙을 실무에 바로 적용할 수 있습니다.

## 액션 모델

| 액션 | 범위 | 대표 도구 |
|---|---|---|
| `read` | 조회 작업 | `list_collections`, `search_documents`, `read_document`, `list_events` |
| `write` | 비파괴 변경 작업 | `create_document`, `update_document`, `move_document`, 댓글/멤버십/lease |
| `delete` | 파괴적 작업 | `delete_document`, `delete_comment`, `delete_collection`, batch delete |
| `admin` | 고급 운영 제어 | 일반 운영에서는 제한적으로 사용 |

## 권장 역할 매핑

| 역할 | 권장 액션 | 이유 |
|---|---|---|
| Reader 에이전트 | `read` | 변경 리스크가 없음 |
| Writer 에이전트 | `read,write` | 생성/수정 가능, 파괴 작업 차단 |
| Operator 에이전트 | `read,write,delete` | 정리/수명주기 작업 담당 |

## 운영 규칙

1. 신규 클라이언트는 항상 `read`로 시작합니다.
2. 검증 통과 후에만 `read,write`로 승격합니다.
3. `delete`는 승인된 운영 경로에만 부여합니다.
4. 장애 회고 시 권한 프로파일을 함께 점검합니다.

## 설정 예시

### Read-only

```bash
OUTLINE_ALLOWED_ACTIONS=read
```

### Read + Write

```bash
OUTLINE_ALLOWED_ACTIONS=read,write
```

### Full Operator

```bash
OUTLINE_ALLOWED_ACTIONS=read,write,delete
```
