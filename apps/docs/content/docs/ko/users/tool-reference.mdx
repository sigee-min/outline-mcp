이 문서는 기능명을 나열하는 대신, 운영 목적에 맞춰 어떤 도구를 선택해야 하는지 바로 판단할 수 있도록 구성했습니다.

| 사용 목적 | 주요 도구 |

| 문서 수명주기 | `delete_document`, `archive_document`, `unarchive_document`, `restore_document`, `list_archived_documents`, `list_trash` |

## Resource URI

모델이 컨텍스트를 직접 읽어야 할 때 다음 URI를 사용할 수 있습니다.

- `outline://collection/{collection_id}`
- `outline://collection/{collection_id}/tree`
- `outline://collection/{collection_id}/documents`
- `outline://document/{document_id}`
- `outline://document/{document_id}/backlinks`

## 운영 팁

- 변경 작업 전에는 조회/구조 도구로 먼저 범위를 확인하세요.
- 배치/수명주기 도구는 소유권과 롤백 전략을 명확히 한 뒤 사용하세요.
- 멀티 에이전트 쓰기에서는 lease와 `safe_update_document`를 함께 사용하세요.
